A Microsoft Defender XDR incident is a correlation of related alerts across endpoints, identities, email, and cloud apps that represents a single attack story. When active incidents accumulate without triage, the threat actor's progress — from initial phishing, to credential theft, to lateral movement, to data staging — is recorded in telemetry but never acted on; the dwell time grows, and an intrusion that could have been contained at the credential stage proceeds to exfiltration and impact. This check confirms there are no active incidents that have not been assigned, classified, and either remediated or resolved within the response window the security team has committed to, so the unified incident view drives action rather than only visibility.
